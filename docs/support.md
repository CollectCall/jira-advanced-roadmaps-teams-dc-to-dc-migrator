# Support

## Maintenance expectations

This is an early-stage public repository. Support is best-effort and prioritizes:

- correctness and safety regressions
- installer and release breakages
- documentation gaps that block safe usage

## Good issue reports

Include:

- Jira edition and version
- source and target environment type
- exact command used
- whether the run was dry-run or apply mode
- sanitized report snippets
- whether ScriptRunner or DB-derived filter CSVs were used

## What maintainers may ask for

- a redacted config snippet
- redacted report JSON or CSV artifacts
- exact CLI version
- a minimal reproduction using sample-like inputs

## Response model

- no SLA is guaranteed
- critical correctness and security issues are prioritized
- feature requests are evaluated for migration safety first, convenience second
